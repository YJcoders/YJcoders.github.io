# 6、url-loader 和 file-loader 结合使用，处理图片

file-loader 可以指定资源文件的位置，以及如何使用版本哈希命名以获得更好的缓存,可以使用相对路径而不用担心 URL 的问题。 配置 name: '[path][name].[ext]', 文件会自动打包在该路径

url-loader 可以根据文件大小，设置转换为内联的 base-64 URL (当文件小于给定的阈值)，这会减少小文件的 HTTP 请求数。如果文件大于该阈值，会调用 fallback 里面的 file-loader 处理。

{
  test: /\.(png|jpe?g|gif|webp)(\?.*)?$/,
  use: [
    {
      loader: 'url-loader',
      options: {
        limit: 10240,
        fallback: {
          loader: 'file-loader',
          options: {
            name: 'img/[name].[hash].[ext]'
          }
        },
      }
    }
  ]
}

# 8、webpack内置插件 DllPlugin

点击查看

DLL(Dynamic Link Library)：动态链接库文件

作用： 把复用性较高的第三方模块（vue, jquery, loadsh等）打包到动态链接库中，在不升级这些库的情况下，动态库不需要重新打包，每次构建只重新打包业务代码

使用：两个配置文件 webpack.config.js (主配置文件) 和 webpack.dll.config.js (用于打包第三方模块配置)

1、webpack.dll.config.js :

const webpack = require('webpack');
// 这里是第三方依赖库 
const vendors = ['vue', 'jquery'];

module.exports = {
  mode: 'production',
  entry: {
    // 定义程序中打包公共文件的入口文件vendor.js 
    vendor: vendors
  },
  output: {
    filename: '[name].[chunkhash].dll.js',
    // 这里是使用将 verdor 作为 library 导出，并且指定全局变量名字是[name]_[chunkhash] 
    library: '[name]_[chunkhash]'
  },
  plugins: [
    new webpack.DllPlugin({
      context: __dirname,
      // 这里是设置 mainifest.json 路径 
      path: 'manifest.json',
      // 此处name  必须和上面的 library 相同
      name: '[name]_[chunkhash]',
    })
  ]
};
