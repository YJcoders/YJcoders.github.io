YJcoders 2018-01-10 Git

# 1、config

# 查看git配置
git config --list
# 配置用户名
git config --global user.name yjcoders
# 配置邮箱
git config --global user.email test@163.com
# 配置密码
git config --global user.password 123
# 配置命令别名
git config --global alias.logs 'log --pretty'
# 自定义设置git log查看输出状态
git config --global alias.xx "log --graph --pretty=format:'%C(auto)%h %C(green)%ad %C(cyan)%cn %C(auto)%d %s %C(green)(%cr)' --date=short"
例：设置alias.logs  那么查看就使用 git logs
# 查看历史命令
git reflog

# 2、branch

# 查看本地分支
git branch
# 查看所有分支，包括远程分支
git branch -a
# 创建新分支
git branch branch-name
# 创建新分支并切换到该分支
git checkout -b branch-name
# 删除分支
git branch -d branch-name
# 强制删除分支
git branch -D branch-name
# 本地分支与远程分支建立关联
git branch --set-upstream-to=origin/test test
# 或者
git branch --set-upstream test origin/test
# 基于当前分支某个节点创建分支
git branch branch-name ac2bsde

# 3、remote

# 查看远程地址
git remote -v
# 添加远程仓库地址，仓库名为origin
git remote add origin git@gitee.com:YJcoders/test1.git
# 更换仓库地址
git remote set-url origin git@gitee.com:YJcoders/test2.git
# 查看本地仓库与远程仓库一一对应关系
git remote show origin
# 删除为origin的远程仓库
git remote rm origin
# 本地分支已经删除，但是 git branch -a 还可以查看到该远程分支（一般多人开发会出现）
# 删除该现象的分支：
git remote prune origin branch-name

# 6、stash

提示： 新增的文件是不会被添加到stash中，只是修改记录会存放

# 当前分支有修改，未提交，需要切换其他分支，需要暂存当前修改记录
git stash
# 回到当前分支，恢复暂存的修改，并删除最近一次的暂存记录
git stash pop
# 查看所有存放记录
git stash list
# 恢复某个记录到工作区
git stash apply stash@{1}
# 删除某个记录
git stash drop stash@{1}
# 删除所有记录
git stash clear

# 7、reset、revert

# 版本回退，commit回退到上一次提交
git reset
# 指定回退到某个节点，工作区未保存的记录，会丢失
git reset --hard acde6sd
# 指定回退到某个节点，工作区和暂存区不受影响
git reset --soft acde6sd
# git rebase 回退到acde6sd节点后，该节点后续提交的记录都会受影响被删除
git push -f   reset之后如果远程仓库也要版本回退，确认无误就强制推送远程仓库
# 撤销某个提交，不影响后续提交
git revert acde6sd
# git revert acde6sd 会生成一个新的提交记录 dd5gfse，历史提交记录不会受影响
# 如果又想恢复上次回退的内容，只需要revert上次回退生成的commit id: git revert dd5gfse

# 8、checkout

# 强制重置某分支到指定commit节点，不能重置当前分支

# 9、rebase

提示：整理未提交的分叉提交历史，为直线；可以对历史提交记录进行合并，删减，修改等

# 基于master分支进行变基，把test分支修改记录同步到master
git rebase master test
# 编辑未完成的rebase操作
git rebase --edit-todo
# rebase过程，出现冲突，解决提交冲突后，继续rebase
git rebase --continue
# 放弃rebase
git rebase --abort
# 对历史提交记录进行操作
git rebase -i ed6sd7d
# git rebase -i HEAD~3 进入vim编辑器，显示最近3次提交
将pick改为对应的：关键字：
s: 合并到上一次提交，显示的最上面（最远的一次）的提交，不能修改为s
p: 选择保留
d: 删除

# 如何dev分支完全覆盖master分支？
1、切换到master分支 2、将本地旧分支master重置为dev: git reset --hard dev  3、推送到远程：git push origin master -f

# git 的一些状态提示

A: 增加的文件.
C: 文件的一个新拷贝.
D: 删除的一个文件.
M: 文件的内容或者mode被修改了.
R: 文件名被修改了。
T: 文件的类型被修改了。
U: 文件没有被合并(你需要完成合并才能进行提交)
X: 未知状态。(很可能是遇到git的bug了，你可以向git提交bug report)

# 问题：git 基于master分支，新创建分支test,直接修改test分支，然后切换到master分支，test分支修改会同步修改到master

# 原因: 因为刚拉取的分支，与原分支的提交是一样的，md5值也是一样，即版本号一致，所以切换分支数据同步修改

解决：其中一个分支，修改后提交一次，使二者版本号不同即可。

# .gitignore 忽略被匹配到的文件

注意： .gitignore只能忽略那些原来没有被track的文件，如果某些文件已经被纳入了版本管理中，则修改.gitignore是无效的
如果不想再提交某个文件，执行：
git rm -r --cached file-name
