# 1、js中有几种数据类型？

1：基础数据类型：Number, String, Boolean, null, undefined, Symbol
2：引用数据类型：Object (包含：Function, Array, Date, RegExp)

# 1.1、检测类型

类型判断：typeof, instanceof, Object.prototype.toString.call()
 typeof 一般用于判断基础数据类型，对于引用数据类型都是object, 无法区分是object还是array等
 类型        typeof              输出
 Number      typeof 1234         number
 String      typeof 'ddd'        string
 Boolean     typeof false        boolean
 null        typeof null         object
 undefined   typeof undefined    undefined
 NaN         typeof NaN          number
 NaN === NaN   false

 Array       typeof [1,2]        object
 Function    typeof function(){} function
 Object      typeof {'a': 1}     object
 Object      typeof Object       'function'   // Object本来就是构造函数，Array, Date也都是function
 Object()    typeof Object()     object

 instanceof 运算符用于检测构造函数的 prototype 属性是否出现在某个实例对象的原型链
 类型        instanceof                输出
 Number     12   instanceof Number     false
 String     'aa' instanceof String     false
 Boolean    true instanceof Boolean    false
            []   instanceof Array      true              // 检测数组方法还有 Array.isArray([])
            ({}) instanceof Object     true

 Object.prototype.toString.call()
 const type = Object.prototype.toString  ,
 类型         type                     输出
 Number       type.call(2)             "[object Number]"
 String       type.call('w')           "[object String]"
 Boolean      type.call(true)          "[object Boolean]"
 Function     type.call(function() {}) "[object Function]"
 Array        type.call([])            "[object Array]"
 null         type.call(null)          "[object Null]"
 undefined    type.call(undefined)     "[object Undefined]"
 Object       type.call({})            "[object Object]"

# 2、js中的变量在内存中的具体存储形式

1：基本数据类型保存在栈内存中，它们的值是系统分配，都有固定大小，通过按值访问
----
注意：闭包中的变量并不保存在栈内存中，而是保存在堆内存中。 这也就解释了函数调用之后之后为什么闭包还能引用到函数内的变量
----
2：引用数据类型保存在堆内存中，值大小不固定，栈内存中存放的该对象的访问地址，此地址指向堆内存中的对象，js不能直接访问堆内存中的数据，实际操作对象时，是操作对象的引用
----
栈内存中变量一般在它的当前执行环境结束就会被销毁被垃圾回收制回收， 而堆内存中的变量则不会，因为不确定其他的地方是不是还有一些对它的引用。 堆内存中的变量只有在所有对它的引用都结束的时候才会被回收。
----

# 3、基本类型对应的内置对象，以及他们之间的装箱拆箱操作

1：装箱：把基本类型转换为对应的引用数据类型
2：拆箱：把引用数据类型转换为基本数据类型

// 装箱，分为显示和隐式装箱
隐式装箱： 每当读取一个基本类型的值时，后台会创建一个该基本类型所对应的对象。在这个基本类型上调用方法，其实是在这个基本类型对象上调用方法。这个基本类型的对象是临时的，它只存在于方法调用那一行代码执行的瞬间，执行方法后立刻被销毁。
var str="hello world";
var strRes=str.split(" ");
字符串本身是不具有对象方法的，但是可以调用这个方法是因为内部做了隐式转换：
1创建String实例：var str = new String('hello world'), 2实例上调用方法var strRes=str.split(" ")，3销毁实例str = null

显示装箱：用内置对象（基本包装对象）例如 Number、Boolean、String 三个基本包装类型（null, undefined 没有） 对基本类型进行显式装箱
var num = new Number(123)
var bool = new Boolean(true)
var Str = new String('123')
通过通过new操作符创建的引用类型的实例，在执行流离开当前作用域之前一直保留在内存中。

// 拆箱
拆箱操作中主要有两个方法，valueOf()方法和toString()方法。这两个方法主要用来检测你返回的是不是一个基本类型的值。
一般是先用valueOf()来检测，如果返回的不是一个基本类型的值，是对象自身，则会继续用toString()来检测，
如果检测结果不是一个基本类型的值，则会报错

1.valueOf() 方法返回指定对象的原始值。
1.toString() 方法返回一个表示该对象的字符串。

let name = new String('sun')
let age = new Number(24)
console.log(typeof name) // object
console.log(typeof age) //  object
// 拆箱操作
console.log(typeof age.valueOf()); // number // 24  基本的数字类型
console.log(typeof name.valueOf()); // string  // 'sun' 基本的字符类型
console.log(typeof age.toString()); // string  // '24' 基本的字符类型
console.log(typeof name.toString()); // string  // 'sun' 基本的字符类型
// 下面这些运算很有意思
[]+[]	//""
{}+{}	//"[object Object][object Object]"
[]+{}	//"[object Object]"
{}+[]	//0

undefined 使一个属性，表示缺少值， 此处应该有值，但是未定义
