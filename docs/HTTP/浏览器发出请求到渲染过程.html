    浏览器发出请求到渲染过程

    vuePress-theme-reco YJcoders    2018 - 2023

    浏览器发出请求到渲染过程

    YJcoders 8/6/2018 HTTP

    # 整体过程

    load 事件: 在整个页面及所有依赖资源如样式表和图片都已完成加载时触发。 DOMContentLoaded 事件：当纯 HTML 被完全加载以及解析完成时触发，不必等待样式表，图片或者子框架完成加载（DOM 树构建完成触发）

    1、发送HTTP请求：地址栏输入域名，开始域名解析，找到对应ip，（如果输入ip就直接）访问地址（建立TCP连接），发送HTTP请求
2、接收返回的HTML文件，开始解析HTML
3、构建DOM树：解析HTML标签，构建成DOM树
4、加载CSS资源：解析HTML，遇到CSS资源，异步加载请求，最终解析合成CSSOM树
5、构建渲染树：DOM树和CSSOM（CSS Object Model）树构建完成之后，合成渲染树
6、进行布局和绘制：浏览器根据渲染树进行布局，计算出每个元素在页面上的大小和位置，然后再绘制到页面

DNS域名解析：
先在浏览器缓存中查询，没有就去本机DNS缓存查询，查询不到再去本机的host里面查询，查询不到再发送DNS查询请求

建立TCP连接，TCP(Transmission Control Protocol)传输控制协议：
在TCP层，有个FLAGS字段，几个标识：SYN, FIN, ACK, PSH, RST, URG.
SYN: synchronous表示建立连接；
FIN: finish表示关闭连接
ACK: acknowledgement表示响应
PSH: push表示有data数据传输
RST: reset表示连接重置
URG: urgent紧急

TCP的3次握手：
  一次握手：客户端发送带有SYN标志的数据包--服务端 （客户端向服务端发送建 立连接请求）
  二次握手：服务端发送带有SYN标志/ACK标志的数据包--客户端 （服务端向客户 端发送信息，表示收到你的信息，可以连接）
  三次握手：客户端发送带有ACK标志的数据包--服务端 （客户端告诉服务端，确 认收到服务器端同意连接的信号）

TCP四次挥手：
  一次：客户端发送FIN标志--服务端（客户端没有数据发送了可以结束连接了）
  二次：服务端发送ACK标志--客户端（服务端表示请求已经收到，等发完数据就关 闭）
  三次：服务端发送FIN标志--客户端（服务端已经发送完成，可以关闭）
  四次：客户端发送ACK标志--服务端（客户端表示收到你的关闭请求，2ms后没有响应，关闭连接）

    # 解析过程

    # css 资源加载，不会阻断 HTML 解析，会阻塞页面的渲染

    会影响后面的js执行
因为：js执行代码可能会依赖到css样式，css只阻塞执行而不阻塞js的加载
css会间接地阻塞js后面资源的加载（css阻塞js，js阻塞其他资源）

DOM树生成之后，要等待CSSOM树生成，然后合并成渲染树，进行布局绘制，完成显示

    # script 标签为什么建议放在 body 标签的底部（defer、async）？

    因为浏览器在渲染html时，遇到js资源时会停止当前的页面渲染，去下载执行js
如果放在头部，文件过大会导致首页白屏时间加长，影响用户体验

    # 重绘（repaint）和回流（reflow）

    回流必将引起重绘，重绘不一定会引起回流
回流比重绘的代价要更高

重绘：当元素样式改变但并不影响它在文档流中的位置时，浏览器重新绘制它的过程称为重绘
例如：color、background-color、visibility

回流：当部分或全部元素的尺寸、结构、或某些属性发生改变时，浏览器重新渲染部分或全部文档的过程称为回流
例如：
浏览器窗口大小发生改变
元素尺寸或位置发生改变
元素字体大小变化
添加或者删除可见的DOM元素
查询或调用某些属性和方法：
clientWidth、clientHeight、clientTop、clientLeft
offsetWidth、offsetHeight、offsetTop、offsetLeft
scrollWidth、scrollHeight、scrollTop、scrollLeft
scrollTo()
scrollIntoView()、scrollIntoViewIfNeeded()
getComputedStyle()
getBoundingClientRect()

减少回流，重绘操作

    # 渲染过程如何优化？

    1、尽量减少对DOM节点的操作，使用DocumentFragment，一次性操作
2、动画、位移操作使用 transform 替代 top
3、动画、位移使用position 的 absolute，fixed属性
4、使用 visibility 替换 display: none ，因为前者只会引起重绘，后者会引发回流（改变了布局）
5、减少css的层级
6、减少首页加载的js和css资源
7、减少重绘和回流（根据对应点优化处理）
8、使用window.requestAnimationFrame()、window.requestIdleCallback()优化批量渲染
