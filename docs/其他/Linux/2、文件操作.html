    # 创建与删除

    1、创建文件
touch test.txt （touch 创建文件 可以创建多个，文件名空格隔开即可）
vim test.txt  创建test.txt文本文件， 并进入编辑器，编辑该文本

2、创建文件夹
mkdir test 在当前目录创建test文件夹
mkdir a/b  创建文件夹a, 并且再a文件夹下创建b文件夹
创建文件夹提示permission denied  可以使用 sudo chmod R 777 /home 添加权限    (home为当前文件夹)

3、删除文件、文件夹
rm demo.txt 删除demo文件
rmdir test 删除test文件夹 （前提是文件夹为空，里面不能有任何文件，即使是空文件夹也不行）
rm -r test 可以删除非空文件夹，不管有没有内容
rm -f demo 强制删除demo文件
rm -rf test 强制删除文件夹，并且没有提示，(谨慎操作)

4、复制与移动
cp -r  test tmp  将test文件夹复制到tmp目录下(tmp目录下会存在test文件夹) -r 表示递归复制test下所有文件
cp -r  test/* tmp  将test文件夹下的内容复制到tmp目录下(tmp目录下只会会存在test下的内容，不会多一层test文件夹)
cp -r  test/. tmp 同上
cp -r * ../tmp 在test文件夹下操作，表示把test下的内容 复制到tmp文件夹下

mv test tmp  将test文件夹移动到tmp目录下(tmp目录下会存在test文件夹)
mv test/* tmp  将test文件夹下的内容移动到tmp目录下(tmp目录下只会会存在test下的内容，不会多一层test文件夹)
mv test test1 将test文件夹重命名为test1（前提test1在当前目录下不存在）
mv * ../tmp 在test文件夹下操作，表示把test下的内容 移动到tmp文件夹下

5、查看文件内容
cat demo  查看当前文件夹下的demo内容， 显示在终端
head demo 查看demo的头部内容
head 5 demo 查看demo 前5行内容
tail demo 查看demo的尾部内容
tail -5 demo 查看demo倒数5行内容
wc -l demo 查看demo文件有多少行

    # 文件的打包，压缩，解压

    常见的压缩方式有：.zip .rar .gz .bz2 .tar .tag.gz .tar.bz2

    # zip
    优点：支持的平台多
缺点：压缩率不高，tar.gz和tar.bz2在压缩率比较好

zip demo.zip demo  压缩demo文件
zip big.zip demo a.zip c.txt  压缩多个文件

unzip demo.zip 解压到当前文件夹
unzip demo.zip -d ./tmp/ 解压到tmp文件夹 （-d指定路径）
unzip a.zip;unzip.zip 同时解压多个压缩包
unzip \*.zip 解压当前下所有的压缩包

unzip -n a.zip (-n 出现同名不覆盖文件，以当前文件为准)
unzip -o a.zip (-o 出现同名,覆盖文件，以压缩包里面的文件内容为准)
unzip -l a.zip (只是查看压缩包里面有哪些文件，不解压)

    # tar
    .tar格式  快且耗存小，因为它只是一个打包工具，并不压缩。
.tar的压缩使用 gzip 或者 bzip2 等其它命令,gzip通常只能处理单个文件，所以一般是选择使用 tar 命令间接的完成解压缩

tar [-参数1][参数f]   （参数1：必选，且一个; 参数f: 最后一个参数，必须）
-c 压缩
-x 解压
-t 查看内容
-r 向压缩文件末尾追加文件
-u 更新原压缩包中的文件

tar [-参数1][参数2][参数f] （参数2：可选）
-z 有gzip属性的
-j 有bz2属性的
-Z 有compress属性的
-v 显示所有过程
-O 将文件解开到标准输出

tar cf test.tar test  将test文件打包为 名字test.tar的打包文件
tar xf test.tar -C test1 将test.tar 解压到test1文件夹下

tar czf test.tar.gz test 将test文件打包并使用gzip方式进行压缩
tar xzf test.tar.gz -C test1 解压
tar cjf test.tar.bz2 及那个test文件打包并使用bzip2进行压缩
tar jxf test.tar.gz -C test1 解压

    # 文件的查找

    1、find  在指定目录下查找
find path -name testfile
find /  -name nginx
find /  -name nginx* #模糊匹配
find /  -mmin -20  #最近20分钟内修改的文件
find /  -mtime -1  #最近1天内修改的文件

2、locate
linux系统每天至少自动扫描一次文件，将结果保存到数据库，locate查的是数据库记录。locate查询比find要快，但最新文件如果在数据库中没有记录,是需要手动更新（updatedb）
locate  config.js

3、which
which从环境变量文件（/etc/profile）中的path目录中查找，且which找的都是可执行文件。
which nginx

4、whereis
与which类似查询可执行文件，whereis查询更多，会查找出相关的man文件
whereis nginx

    # 文本的查找

    /正则  ?正则
/test 向后查找文中test 字段
?test 向前查找
查找到之后，n 命令查找下一个
:s/test/TEST 针对当前行 查找到的第一个test字符进行替换为TEST
:%s/test/TEST 针对整个文本进行查找
:%s/test/TEST/g 对文本所有的test字段进行替换
:3,5s/test/TEST 针对第3到5行的字符进行替换

:set nu 显示行号，只对当次生效
:set nonu 隐藏行号，只对当次生效
永久生效需要修改 etc/vimrc 配置文件： vim /etc/vimrc 编辑该文件，尾行添加 set nu 然后保存

可视模式：
小写v 字符可视模式，移动光标选择字符
大写V 行可视模式，移动光标，选择以行为单位
ctrl + v 块模式，选择一块
