    创建对象

    YJcoders 10/15/2019 JS

    通过对象字面量和 Object 构造函数可以创建对象，但使用同一个接口创建大量对象时，会重复很多代码 为解决问题，出现了以下创建对象方式

    # 字面量和系统构造函数

    1、字面量
var obj = {
  name: 'zs',
  age: 16,
  say: function () {
    console.log(this.name)
  }
}

2、new Object()
var obj = new Object();
obj.name = "zs";
obj.age = 30;
obj.say = function () {
    console.log("say");
};

    # 1、工厂模式

    function createObj (name, age) {
  var obj = new Object()
  obj.name = name
  obj.age = age
  obj.say = function () {
    console.log('say')
  }
  return obj
}

var obj1 = createObj('zs', 14)
console.log(obj1)
var obj2 = createObj('ls', 16)

---工厂模式可以重复调用这个工厂函数,工厂模式虽然解决了创建多个相似对象的问题，
---但是没有解决对象识别问题，即不能知道一个对象的类型。

    # 2、自定义构造函数模式

    function Person(name,age){
    this.name=name;
    this.age=age;
    this.say=function(){
        console.log("name");
    };
}
var per = new Person("zs", 18);

--- 可以将他的实例标识为一种特定的类型
--- 但是，构造函数的每个方法都会再每个实例上重新创建一次

    # 3、原型模式

    function Person() {
}
Person.prototype.name = 'zs'
Person.prototype.sayName = function() {
  console.log(this.name)
}

var per1 = new Person()

--- 可以让所有对象实例共享它所包含的属性和方法
--- 但是引用类型的属性共享，一个修改，都会受影响

    # 4、组合使用构造函数模式和原型模式

    function Person(name,age){
    this.name=name;
    this.age=age;
}
Person.prototype.sayName = function() {
  console.log(this.name)
}

var per1 = new Person('zs', 14)

--- 构造函数模式用于定义实例属性，原型模式用于定义方法和共享的属性，这样每个实例都有自己的一份实例属性的副本，
又同时共享着对方法的引用，最大限度的节省了内存
