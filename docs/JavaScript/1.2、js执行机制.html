    # 1、js 是单线程的，同步和异步执行

    JavaScript的主要用途是与用户互动，以及操作DOM。这决定了它只能是单线程，否则会带来很复杂的同步问题。
比如，假定JavaScript同时有两个线程，一个线程在某个DOM节点上添加内容，另一个线程删除了这个节点，这时如果是多线程可能就会出问题

    # 4、同步任务和异步任务

    所有任务可以分成两种，一种是同步任务（synchronous），另一种是异步任务（asynchronous）。
同步任务指的是，在主线程上排队执行的任务，只有前一个任务执行完毕，才能执行后一个任务；
异步任务指的是，不进入主线程、而进入"任务队列"（task queue）的任务，只有"任务队列"通知主线程，某个异步任务可以执行了，该任务才会进入主线程执行

"任务队列"是一个先进先出的数据结构

    # 4.1、事件循环

    同步的进入主线程，异步的进入Event Table（事件列表）并注册函数。
1、当Event Table中指定的事情完成时，会将这个函数移入Event Queue。
2、主线程内的任务执行完毕为空，会去Event Queue读取对应的函数，进入主线程执行。
3、上述过程会不断重复，也就是常说的Event Loop(事件循环)。

    # 5、异步任务分为：宏任务与微任务

    macro-task(宏任务)：包括整体代码script，setTimeout，setInterval
micro-task(微任务)：Promise，process.nextTick

进入整体代码(宏任务)后，开始第一次循环。接着执行所有的微任务。然后再次从宏任务开始，找到其中一个任务队列执行完毕，再执行所有的微任务

    # 6、浏览器中示例

    // 第1执行
console.log("1");

// 异步任务，宏任务，进入事件列表执行
setTimeout(function () {
  console.log("2");
  // process.nextTick(function() {
  //     console.log('3');
  // })
  new Promise(function (resolve) {
    console.log("4");
    resolve();
  }).then(function () {
    console.log("5");
  });
});

// 异步任务，微任务，直接进入任务队列
// process.nextTick(function() {
//     console.log('6');
// })

new Promise(function (resolve) {
  // 第2执行
  console.log("7");
  resolve();
  // 异步任务，微任务，直接进入任务队列
}).then(function () {
  console.log("8");
});

// 异步任务，宏任务，进入事件列表执行
setTimeout(function () {
  console.log("9");
  // process.nextTick(function() {
  //     console.log('10');
  // })
  new Promise(function (resolve) {
    console.log("11");
    resolve();
  }).then(function () {
    console.log("12");
  });
});

// 第一轮事件循环执行： 1 、 7
// 主线程空闲，读取异步执行已经完成的任务队列中的任务，执行： 6 、8
// 第二轮事件循环，执行第一个异步宏任务setTimeout, 执行： 2 、4，
// 宏任务执行完成，执行微任务，执行： 3、 5
// 第三轮事件循环，执行第二个异步宏任务setTimeout, 执行： 9、  11
// 宏任务执行完成，执行微任务，执行： 10、 12

// 综上：1、7、6、8、2、4、 3、5、9、11、10、12
