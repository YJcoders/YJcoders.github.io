    预加载、异步加载

    YJcoders 10/28/2022 HTTP

    # 资源加载优先级

    # 预加载（preload、prefetch、dns-prefetch）

    # 1、preload 对当前资源进行预加载，无需等解析到 script 或者 link 标签时才加载，优先级更高 high

    // preload <link>标签 属性rel的值
// 用于肯定会使用到的资源，页面渲染的时候就会加载
// 仅预加载资源，不会立即执行
<link rel="preload" href="style.css" as="style"/>
<link rel="preload" href="index.js" as="script"/>

// 注意
对于跨域的文件使用 preload 时，必须加上 crossorigin 属性，否则会发送两个请求（重复请求）
<link rel="preload" as="font" crossorigin href="https://at.alicdn.com/t/font_zck90zmlh7hf47vi.woff">

// 其他可以被 preload的as属性
audio: 音频文件。
document: 一个将要被嵌入到或内部的HTML文档。
embed: 一个将要被嵌入到元素内部的资源。
fetch: 那些将要通过fetch和XHR请求来获取的资源，比如一个ArrayBuffer或JSON文件。
font: 字体文件。
image: 图片文件。
object: 一个将会被嵌入到元素内的文件。
script: JavaScript文件。
style: 样式表。
track: WebVTT文件。
worker: 一个JavaScript的web worker或shared worker。
video: 视频文件。

    # 2、prefetch 对当前资源进行预加载，但是是在浏览器空闲的时候加载，优先级低 Lowest

    // prefetch <link>标签 属性rel的值
// 用于可能会用到的资源，浏览器空闲的时候加载
// 仅预加载资源，不会立即执行
<link rel="prefetch" href="text.css" />
<link rel="prefetch" href="text.js" />

    # 3、dns-prefetch 在浏览器空闲的时候，进行 DNS 预解析

    // 根据域名找到相应IP，缓存到系统缓存中
// DNS解析会有延迟，对于移动网络更为明显，使用dns-prefetch可以缩短解析时间
<link rel="dns-prefetch" href="https://baidu.com/">

当浏览器访问一个域名的时候，需要解析一次DNS，获得对应域名的ip地址。
在解析过程中，按照 浏览器缓存 -> 系统缓存 -> 路由器缓存 -> ISP(运营商)DNS缓存 -> 根域名服务器 -> 顶级域名服务器 -> 主域名服务器
的顺序逐步读取缓存，直到拿到IP地址

// 注意
1、dns-prefetch 仅对跨域域上的 DNS查找有效，因此请避免使用它来指向相同域。这是因为，到浏览器看到提示时，您站点域背后的IP已经被解析
2、将 dns-prefetch 与 preconnect(预连接) 结合使用
由于dns-prefetch 仅执行 DNS查找，而 preconnect 会建立与服务器的连接。
如果站点是通过HTTPS服务的，两者的组合会涵盖DNS解析，建立TCP连接以及执行TLS握手。将两者结合起来可进一步减少跨域请求的延迟
// 例：
<link rel="preconnect" href="https://fonts.gstatic.com/" crossorigin>
<link rel="dns-prefetch" href="https://fonts.gstatic.com/">

3、如果页面需要建立与许多第三方域的连接，就不用使用preconnect。 preconnect 最好仅用于最关键的连接，其他的只使用dns-prefetch即可

    # 异步加载（defer、async、module）

    # 1、defer 加载资源和渲染 dom 并行，但会等到 HTML 解析完成之后执行

    // DOMContentLoaded 事件触发之前执行
// defer 会按顺序执行脚本
<script defer src="index.js"></script>
    1
    2
    3

    # 1、module 异步加载 ES6 模块，不会造成阻塞，等到整个页面渲染完，再执行模块脚本

    <script type="module" src="index.js"></script>
    1

    # 3、async 加载资源和渲染 dom 并行，加载完成后，暂停 HTML 解析，立即执行 js 脚本

    // async 资源加载完成就执行，无法保证顺序
<script async src="index.js"></script>
