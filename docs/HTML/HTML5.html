    # H5 的新特性有哪些？

    1、语义化新标签: header, nav, aside, section, article, footer
2、画布API: canvas
3、音频、视频API: audio, video
4、拖放API: drag
  - 拖放源对象的API: dragstart: 拖放开始；drag: 拖放中；dragend: 拖放结束
  - 目标对象的API: dragenter: 拖放进入；dragover: 拖放悬停；dragleave: 拖放离开；drop: 拖放结束
  dataTransfer: 数据传递对象
  - 源对象使用： e.dataTransfer存储数据：e.dataTransfer.setData(key, value)
  - 目标对象使用： e.dataTransfer获取数据：e.dataTransfer.getData(key)
5、input属性新增：tel, email, number, datatime等
6、地理定位API: geolocation
7、 Web Storage: localStorage, sessionStorage
8、WebSocket: 客户端和服务端通信的一个新的持久化的协议
9、Web Worker：为 JavaScript 创造多线程环境，允许主线程创建 Worker 线程，将一些任务分配给后者运行。在主线程运行的同时，Worker 线程在后台运行，两者互不干扰。等到 Worker 线程完成计算任务，再把结果返回给主线程

    # 对 web 标准以及 w3c 的理解与认识?

    web标准：分为三部分：结构、表现、行为；对应的就是html标签布局、css、js
W3C对web标准提出规范化要求，也就是代码规范：
1：对结构要求：（标签规范有利于搜索引擎搜索，利于SEO）
标签要闭合、标签要小写，不能随意嵌套，（p标签不能嵌套块级元素，会被块级元素分为两个p标签）
2：对css：尽量少使用行内样式，尽量统一使用类名，便于解析，提高速度，类名要见名之意
3：使用外部链接css、js使其分离，页面加载速度更快

    # 回流与重绘

    回流：受影响的部分，页面布局重新构建
重绘：页面元素属性修改，引起重绘
回流必定引起重绘，重绘不一定引起回流；回流消耗性能，尽量避免回流
会引起回流的操作：
调整窗口大小；DOM操作，增减，位置变化；元素属性变化：margin,padding,border,width,height
获取页面一些关于窗口的属性：offsetTop、offsetLeft、 offsetWidth、offsetHeight、scrollTop、scrollLeft、scrollWidth、scrollHeight、 clientTop、clientLeft、clientWidth、clientHeight；因为浏览器为了获取正确的属性，需要经过回流，重新获取；不过我们可以对这些属性进行缓存，不直接获取
