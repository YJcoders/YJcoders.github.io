    # 1、typeof 用来获取一个变量或对象（包括函数）的类型

    对象转类型 语法：类型 = typeof 实例对象

    function add(first: string | number, second: string | number) {
  if (typeof first === "string" || typeof second === "string") {
    return `${first}${second}`;
  }
  return first + second;
}

// 对基本类型使用
let s = "hello";
let n: typeof s; // let n: string

// 对对象使用
const dog = { name: "牧羊犬", color: "黑色" };
type Dog = typeof dog;
// type Dog = {
//   name: string;
//   color: string;
// }

// 对函数使用
function identity<Type>(arg: Type): Type {
  return arg;
}
type Result = typeof identity;
// type Result = <Type>(arg: Type) => Type

// 对枚举类型enum使用
// 在 TypeScript 中，枚举类型是一种特殊的类型，它会被编译成普通的 JavaScript 对象
enum HttpMethod {
  Get,
  Post,
}
// 会被编译为
("use strict");
var HttpMethod;
(function (HttpMethod) {
  HttpMethod[(HttpMethod["Get"] = 0)] = "Get";
  HttpMethod[(HttpMethod["Post"] = 1)] = "Post";
})(HttpMethod || (HttpMethod = {}));

console.log(HttpMethod);

// 对 enum 类型只使用 typeof 一般没什么用，通常还会搭配 keyof 操作符用于获取属性名的联合字符串
type Method = keyof typeof HttpMethod; // type Method = "Get" | "Post"

// 在使用 typeof 操作符的过程中，如果想要获取更精确的类型，可以结合 TypeScript 3.4 版本中引入的 const 断言
let requestMethod = "Get";
let requestMethod2 = "Get" as const;
type R0 = typeof requestMethod; // string
type R1 = typeof requestMethod2; // "Get"

let dog1 = {
  name: "泰迪",
  color: "red",
};
let dog2 = {
  name: "柯基",
  color: "white",
} as const;
type T1 = typeof dog1;
// type T1 = {
//   name: string;
//   color: string;
// }
type T2 = typeof dog2;
// type T2 = {
//   readonly name: "柯基";
//   readonly color: "white";
// }

    # 2、instanceof 用于检查一个值是否是给定构造函数或类的实例

    class Foo {
  foo = 123;
}
class Bar {
  bar = 123;
}
function test(arg: Foo | Bar) {
  if (arg instanceof Foo) {
    console.log(arg.foo); // 正常
    console.log(arg.bar); // 报错 类型“Foo”上不存在属性“bar”。ts(2339)
  } else {
    // 一定是 'Bar'
    console.log(arg.foo); // 类型“Bar”上不存在属性“foo”。ts(2339)
    console.log(arg.bar); // 正常
  }
}
test(new Foo());
test(new Bar());

    # 3、keyof 用来获取对象中所有 key 值

    获取类型属性键 语法：类型 = keyof 类型

    // 获取一个类型所有键值，返回一个联合类型
interface Person {
  name: string;
  age: number;
  sex: string;
}
type Keys = keyof Person; // Keys 类型为 'name' | 'age' | 'sex' 组成的联合类型

// 如果这个类型有一个 string 或者 number 类型的索引签名，keyof 则会直接返回这些类型
// 数字索引 -> keyof StringArray1 => number
type StringArray1 = { [n: number]: unknown };
type A = keyof StringArray1;
// type A = number

// 字符串索引 -> keyof StringArray => string | number
type StringArray1 = { [k: string]: boolean };
type M = keyof StringArray1;
// type M = string | number
//  JavaScript 对象的属性名会被强制转为一个字符串，所以 obj[0] 和 obj["0"] 是一样的

// 限制访问对象的 key 合法化
function getProperty<Type, Key extends keyof Type>(obj: Type, key: Key) {
  return obj[key];
}
let x = { a: 1, b: 2, c: 3, d: 4 };
getProperty(x, "a");
getProperty(x, "m"); // 报错 类型“"m"”的参数不能赋给类型“"a" | "b" | "c" | "d"”的参数

    # 4、in 用于遍历操作

    用于检查对象是否具有特定的属性，只能用在类型的定义中 语法：[ 自定义变量名 in 枚举类型 ]: 类型

    // 用于基础类型 string, number, symbol
type StringKey = {
  [key in string]: any;
};
// type StringKey = {
//     [x: string]: any;
// }
type NumberKey = {
  [key in number]: any;
};
// type NumberKey = {
//     [x: number]: any;
// }
type SymbolKey = {
  [key in symbol]: any;
};
// type SymbolKey = {
//     [x: symbol]: any;
// }
interface PersonArray {
  [index: number]: Person;
}
interface PlainObject {
  [key: string]: any;
}

// 用于联合类型
type Property = "name" | "age";
type Obj = {
  [k in Property]: k;
};
// type Obj = {
//   name: "name";
//   age: "age";
// }

type ToString<T> = {
  [key in keyof T]: string;
};

// 用于枚举类型
enum Map {
  A,
  B,
  C,
}
type iMap = {
  [key in Map]: string;
};
// type iMap = {
//   0: string;
//   1: string;
//   2: string;
// }

    语法：参数 is 类型

    // 验证是否是number类型
function isNumber(x: any): x is number {
  return typeof x === "number";
}
// 验证是否是string类型
function isString(x: any): x is string {
  return typeof x === "string";
}

function doSometing(value: string | number) {
  if (isString(value)) {
    // TS 可以识别这个分支中 value 是 string 类型的参数
  } else {
    // TS 可以识别这个分支中 value 是 number 类型的参数
  }
}

    # 6、extends(类型约束)

    范围限制和条件判断

    // 1、接口继承
interface T1 {
  name: string;
}
interface T2 {
  sex: number;
}
// 多重继承，逗号隔开
interface T3 extends T1, T2 {
  age: number;
}
const t3: T3 = {
  name: "test",
  sex: 1,
  age: 18,
};

// 2、条件判断
// 如果extends前面的类型能够赋值给extends后面的类型，那么表达式判断为真
// 如果 T 可以满足类型 Person 则返回 Person 类型，否则为 T 类型
type IsPerson<T> = T extends Person ? Person : T;

// 3、泛型用法
type P<T> = T extends "x" ? string : number;
type A3 = P<"x" | "y">; // A3的类型是 string | number
// 使用extends关键字的条件类型（即上面的三元表达式类型），如果extends前面的参数是一个泛型类型，当传入该参数的是联合类型，则使用分配律计算最终的结果。分配律是指，将联合类型的联合项拆成单项，分别代入条件类型，然后将每个单项代入得到的结果再联合起来，得到最终的判断结果

// 4、特殊的never
// never是所有类型的子类型
type A1 = never extends "x" ? string : number; // string

type P<T> = T extends "x" ? string : number;
type A2 = P<never>; // never  此处 never被认为是空的联合类型

// 5、防止条件判断中的分配
// 在条件判断类型的定义中，将泛型参数使用[]括起来，即可阻断条件判断类型的分配，此时，传入参数T的类型将被当做一个整体，不再分配
type P<T> = [T] extends ["x"] ? string : number;
type A1 = P<"x" | "y">; // number
type A2 = P<never>; // string

    # 7、infer 声明特定位置待推断类型

    表示在 extends 条件语句中以占位符出现的用来修饰数据类型的关键字，被修饰的数据类型等用到的时候才能被推断出来

    // infer 只能在条件类型的 extends 子句中使用，同时 infer 声明的类型变量只在条件类型的 true 分支中可用
type Wrong1<T extends (infer U)[]> = T[0]; // Error
type Wrong2<T> = (infer U)[] extends T ? U : T; // Error
type Wrong3<T> = T extends (infer U)[] ? T : U; // Error

// 1、获取函数参数类型  出现位置在参数上
type UnpackedArray<T> = T extends (infer U)[] ? U : T;
type U0 = UnpackedArray<number[]>; // type U0 = number

// 例
type User = {
  id: number;
  name: string;
};
type PropertyType<T> = T extends { id: infer U; name: infer R } ? [U, R] : T;
type U3 = PropertyType<User>; // [number, string]

// 2、获取函数类型的返回值类型  出现位置再返回值上
type ReturnType1<T> = T extends (...args: any[]) => infer P ? P : any;
type Func = () => string;
type Return = ReturnType1<Func>; // type Return = string

// 在给限定了函数类型的变量赋值函数时，赋值给变量的函数，其参数可以是比变量类型定义里要求的属性更少(逆变)，返回值可以是比变量类型定义里要求的属性更多(协变)。
// 用于赋值的函数的参数的属性必须比被赋值的变量的要求更少,用于赋值的函数的返回值的属性必须比被赋值的变量的要求更多
// 3、逆变  函数的参数类型兼容性是典型的逆变
// 当比较两个函数类型是否兼容时，函数参数为逆变
let fn1!: (a: string, b: number) => void;
let fn2!: (a: string, b: number, c: boolean) => void;
fn2 = fn1; // 正确
fn1 = fn2; // 报错 不能将类型“(a: string, b: number, c: boolean) => void”分配给类型“(a: string, b: number) => void”。ts(2322)

// 在逆变位置上，若同一个类型变量存在多个候选者，则最终的类型将被推断为交叉类型
type Bar<T> = T extends { a: (x: infer U) => void; b: (x: infer U) => void }
  ? U
  : never;
type U5 = Bar<{ a: (x: string) => void; b: (x: number) => void }>; // never(同时为string和number类型： string && nuber，就是never)

// 4、协变
let fn1!: (a: string, b: number) => string;
let fn2!: (a: string, b: number) => string | number | boolean;
fn2 = fn1; // 正确
fn1 = fn2; // 报错 不可以将 string|number|boolean 赋给 string 类型

type PropertyType<T> = T extends { id: infer U; name: infer U } ? U : T;
type User = {
  id: number;
  name: string;
};
type U4 = PropertyType<User>; // string | number
// {id: infer U, name: infer U} 中的两个infer U都处于协变位置， 在协变位置上，若同一个类型变量存在多个候选者，则最终的类型将被推断为联合类型
